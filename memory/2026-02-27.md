## Design Decisions

### 1. Slack as Trigger (Not Direct Webhook)
**Why:** Easier debugging, Todd can monitor in real-time, simpler for MVP.

**Pros:**
- See messages visually
- Can manually post test leads
- No need for public webhook endpoint
- OpenClaw Slack integration already exists

**Cons:**
- Slightly slower than direct webhook (still <5 sec)
- Requires Slack credentials

**Decision:** Use Slack. Speed difference negligible, debugging value huge.

### 2. Zapier for SMS (Not Direct Twilio API)
**Why:** Simpler, no credential management, Todd already uses Zapier.

**Pros:**
- No Twilio API keys in code
- Can swap SMS providers without code changes
- Zapier handles retries
- Less code to maintain

**Cons:**
- Extra hop in architecture (adds latency)
- Requires Zapier paid plan ($20-50/mo)

**Decision:** Use Zapier. $20/mo cost is acceptable, simplicity is worth it for MVP.

### 3. Stateless for MVP (No Database)
**Why:** Keep it simple for first client. Add persistence in Phase 2 if needed.

**Pros:**
- No DB to set up/maintain
- Faster to build and deploy
- Lower monthly costs

**Cons:**
- If Hermes restarts mid-conversation, conversation is lost
- Can't do analytics without storage

**Mitigation:**
- Write active conversations to WORKING.md before shutdown
- Resume on restart
- Phase 2: Add DynamoDB or Redis if needed

**Decision:** Stateless for MVP. Monitor for conversation loss issues.

## Technical Specs

### Customer Data Parser
**Input format** (expected from Zapier):
```
New Lead: John Doe from Acme Corp
Phone: +15551234567
Email: john@acme.com
Company: Acme Corp
```

**Parsing strategy:**
- Regex to extract phone (E.164 format: +1XXXXXXXXXX)
- Regex to extract email (basic validation)
- Extract name from first line
- Extract company (optional field)
- Handle variations (missing fields, different formats)

**Validation:**
- Phone must be valid US mobile number (not landline)
- Email must have @ and domain
- Name required (if missing, use "there" as fallback)

### SMS Message Templates
**Tone:** Casual, friendly, one question at a time

**Question 1 (Service Type):**
```
"Hey {name}! Got your request for {service}. Quick question — is this for home or business? Reply 1 for Home, 2 for Business."
```

**Question 2 (Urgency):**
```
"Got it! How soon do you need this done? Reply 1 for ASAP, 2 for Next 2-4 weeks, 3 for Just exploring."
```

**Question 3 (Budget):**
```
"Perfect. What's the approximate size of the job? Reply 1 for Small (under $500), 2 for Medium ($500-$2k), 3 for Large ($2k+)."
```

**Question 4 (Decision Maker):**
```
"Last question: Are you the decision maker? Reply 1 for I can decide, 2 for Need to check with someone, 3 for Need to get bids."
```

**Completion:**
```
"Thanks {name}! We've got all we need. Our team will reach out within 4 hours. Talk soon!"
```

### Lead Scoring Formula
```javascript
score = 50 (baseline)

// Service type (max +15)
if (serviceType === 'commercial') score += 15
if (serviceType === 'residential') score += 10

// Urgency (max +30, most important)
if (urgency === 'asap') score += 30
if (urgency === 'next_month') score += 15
if (urgency === 'exploring') score -= 10

// Budget (max +20)
if (budget === 'over_2k') score += 20
if (budget === '500_2k') score += 10
if (budget === 'under_500') score += 5

// Decision authority (max +15)
if (decisionMaker === 'yes') score += 15
if (decisionMaker === 'need_approval') score += 5
if (decisionMaker === 'getting_bids') score -= 5

// Clamp to 0-100
score = Math.max(0, Math.min(100, score))
```

**Score tiers:**
- 80-100: 🔥 Hot Lead (call within 1 hour)
- 60-79: 🔶 Warm Lead (call within 4 hours)
- 40-59: 🔵 Cool Lead (call within 24 hours)
- 0-39: ❄️ Cold Lead (low priority, add to nurture)

### Zapier Webhook Payload
**POST to:** `process.env.ZAPIER_WEBHOOK_URL`

**Format:**
```json
{
  "phone_number": "+15551234567",
  "message": "Hey John! Quick question...",
  "lead_id": "copilot_12345",
  "lead_score": 85,
  "lead_tier": "hot",
  "customer_name": "John Doe",
  "timestamp": "2026-02-27T16:30:00Z"
}
```

**Retry logic:**
- 3 attempts max
- Exponential backoff: 1s, 2s, 4s
- If all fail: alert Todd in Slack

## Challenges Anticipated

### 1. Parsing Variability
**Problem:** Zapier message format might vary (different field names, missing data)

**Solution:** 
- Write flexible regex (multiple patterns)
- Log unparseable messages for manual review
- Fallback to sensible defaults (name="there", company=null)

### 2. SMS Deliverability
**Problem:** Some leads might have landlines, invalid numbers, or international numbers

**Solution:**
- Validate phone format (E.164)
- Check if mobile (basic carrier lookup if needed)
- Skip if invalid, log for Todd to review

### 3. Conversation State Loss
**Problem:** If Hermes restarts mid-conversation, lead might get duplicate messages

**Solution:**
- Write active conversations to WORKING.md
- On startup, read WORKING.md and resume
- Add deduplication check (don't message same lead twice)

### 4. Error Handling
**Problem:** Zapier webhook might fail, Twilio might be down

**Solution:**
- Retry 3x with backoff
- If still fails, alert Todd in Slack
- Log detailed error for debugging
